## 1. INTRODUCCIÓN

### 1.1 Propósito
TubeMassDL es una aplicación de escritorio para Windows que permite capturar enlaces de YouTube y otros sitios (Vimeo, TikTok, Instagram, etc.) de forma automática mediante un **listener del portapapeles**, acumularlos en una **cola de descargas**, y procesarlos masivamente. La aplicación está diseñada para ser **"abrir, copiar enlaces, descargar"** y no requiere instalación de servicios ni procesos en segundo plano al cerrarse.

### 1.2 Objetivos Principales
1. **Captura automática de enlaces**: Detectar silenciosamente enlaces copiados al portapapeles.
2. **Cola de descargas**: Acumular enlaces para descarga masiva posterior.
3. **Descarga inteligente**: Usar `yt-dlp` para sitios multimedia (YouTube, Vimeo, etc.) y HTTP para archivos directos.
4. **No intrusivo**: Funciona en segundo plano cuando está minimizada, parpadea en la barra de tareas al capturar nuevos enlaces.
5. **Cero residuos**: Al cerrar la aplicación, **todo** se cierra (sin servicios, sin procesos ocultos).

### 1.3 Flujo de Trabajo del Usuario
```
1. Usuario abre TubeMassDL.
2. La aplicación se minimiza a la bandeja (opcional).
3. Usuario navega por páginas web copiando enlaces (Ctrl+C).
4. La aplicación captura los enlaces silenciosamente y los agrega a la cola.
5. El icono parpadea en la barra de tareas para notificar novedades.
6. Usuario abre la aplicación, revisa la cola, selecciona enlaces y presiona "Descargar Todos".
7. La aplicación procesa la cola y guarda los archivos en la carpeta seleccionada.
8. Usuario cierra la aplicación → **el listener se desactiva inmediatamente**.
```

## 3. REQUISITOS NO FUNCIONALES

### 3.1 RNF-01: Rendimiento
- La UI no debe congelarse durante las descargas (uso de `async/await` y `Task`).
- El listener del portapapeles debe tener un impacto mínimo en el rendimiento del sistema.
- Uso de memoria < 200MB.

### 3.2 RNF-02: Usabilidad
- "Abres la app, copias enlaces, descargas." Sin configuración compleja.
- Feedback visual inmediato: ícono parpadea, notificaciones, progreso en la UI.
- Botones grandes y claros.

### 3.3 RNF-03: Cero Residuos
- Al cerrar la aplicación, **no quedan procesos en segundo plano**.
- No se registra en el inicio de Windows.
- No se instalan servicios.

### 3.4 RNF-04: Compatibilidad
- Windows 10/11 (x64).
- .NET 8 Runtime (empaquetado).
- Firefox o Chrome para cookies (para sitios que requieran autenticación).

### 4.4 Flujo de Datos (Secuencia de Captura)

```
1. Usuario copia enlace (Ctrl+C)
   ↓
2. ClipboardMonitor detecta cambio (cada 500ms)
   ↓
3. Valida que sea URL y sea nueva
   ↓
4. Dispara evento LinkCaptured
   ↓
5. LinkCollector recibe el evento
   ↓
6. Agrega item a la cola (ObservableCollection)
   ↓
7. Notifica a la UI (QueueUpdated)
   ↓
8. UI actualiza el ListBox y el contador
   ↓
9. SystemTrayService inicia parpadeo
   ↓
10. Muestra notificación emergente (balloon tip)
   ↓
11. En segundo plano, intenta obtener información del enlace (título, fuente)
```

### 4.5 Flujo de Datos (Secuencia de Descarga)

```
1. Usuario selecciona items en la cola
   ↓
2. Presiona "DESCARGAR TODOS"
   ↓
3. DownloadManager recibe la lista de URLs
   ↓
4. Para cada URL:
   a. SiteDetector detecta el tipo de sitio
   b. Si es sitio multimedia (YouTube, Vimeo, etc.) → YtDlpDownloader
   c. Si es archivo directo → HttpDownloader
   d. Si es desconocido → intenta con yt-dlp primero, fallback a HTTP
   ↓
5. Actualiza progreso en cada item
   ↓
6. Marca como Completado o Error
   ↓
7. Notifica finalización
```

## 6. ESPECIFICACIÓN DE SERVICIOS

### 6.1 ClipboardMonitor

**Propósito:** Monitorear el portapapeles en segundo plano.

| Método | Parámetros | Retorno | Descripción |
|--------|------------|---------|-------------|
| `Start()` | - | `void` | Activa el monitoreo del portapapeles |
| `Stop()` | - | `void` | Desactiva el monitoreo |
| `Dispose()` | - | `void` | Libera recursos y detiene el timer |

**Eventos:**
- `LinkCaptured`: Se dispara cuando se captura un nuevo enlace válido.

**Comportamiento:**
- Verifica el portapapeles cada 500ms.
- Solo captura URLs http/https.
- Ignora enlaces duplicados.
- No bloquea la UI.

---

### 6.2 LinkCollector

**Propósito:** Gestionar la cola de enlaces capturados.

| Método | Parámetros | Retorno | Descripción |
|--------|------------|---------|-------------|
| `OnLinkCaptured()` | `CapturedLink` | `void` | Agrega un enlace a la cola |
| `ClearCompleted()` | - | `void` | Elimina items completados o con error |
| `ClearAll()` | - | `void` | Vacía toda la cola |
| `GetSelectedItems()` | - | `List<DownloadQueueItem>` | Obtiene items seleccionados |

**Eventos:**
- `QueueUpdated`: Se dispara cuando la cola cambia.

---

### 6.3 SystemTrayService

**Propósito:** Gestionar el icono en la bandeja del sistema.

| Método | Parámetros | Retorno | Descripción |
|--------|------------|---------|-------------|
| `ShowNotification()` | `title, message, timeout` | `void` | Muestra notificación emergente |
| `StartBlinking()` | - | `void` | Inicia parpadeo del icono |
| `StopBlinking()` | - | `void` | Detiene parpadeo |
| `Dispose()` | - | `void` | Libera recursos |

**Menú Contextual:**
- "📋 Mostrar Cola": Restaura la ventana.
- "⏸️ Pausar Captura": Pausa/Reanuda el listener.
- "---": Separador.
- "🚪 Salir": Cierra la aplicación.

---

### 6.4 DownloadManager

**Propósito:** Orquestar la descarga de enlaces.

| Método | Parámetros | Retorno | Descripción |
|--------|------------|---------|-------------|
| `DownloadAsync()` | `url, outputPath, format` | `Task<DownloadResult>` | Descarga una URL |
| `GetLinkInfoAsync()` | `url` | `Task<LinkInfo>` | Obtiene información del enlace |
| `Pause()` | - | `void` | Pausa la descarga actual |
| `Resume()` | - | `void` | Reanuda la descarga |
| `Stop()` | - | `void` | Detiene la descarga |

**Eventos:**
- `ProgressUpdated`: Progreso de descarga.
- `VideoCompleted`: Video completado o con error.
- `AllCompleted`: Todas las descargas completadas.
- `Log`: Mensajes de log.

---

### 6.5 YtDlpDownloader

**Propósito:** Descargar contenido multimedia usando `yt-dlp`.

**Comandos utilizados:**

**Carga de lista:**
```
yt-dlp --flat-playlist --dump-json --no-warnings --ignore-errors "{url}"
```

**Descarga (con antibloqueo):**
```
yt-dlp --cookies-from-browser chrome
       --sleep-interval {15-45}
       --max-sleep-interval {45-90}
       --limit-rate 5M
       --wait-for-video 30
       --retries 5
       --fragment-retries 5
       --no-mtime
       -f "{formato}"
       -o "{outputPath}\%(title)s.%(ext)s"
       --no-playlist
       --progress
       --newline
       "{url}"
```

## 7. MANEJO DE ERRORES

### 7.1 Errores Comunes y Soluciones

| Error | Causa | Solución |
|-------|-------|----------|
| `DPAPI` | Chrome cambió encriptación | Usar Firefox o exportar cookies manualmente |
| `n challenge` | Falta Node.js/EJS | Instalar Node.js y ejecutar `npm install -g ejs` |
| `Requested format not available` | Formato no disponible | Usar fallback a menor calidad |
| `HTTP 429` | Demasiadas peticiones | Activar modo antibloqueo |
| `Acceso denegado al portapapeles` | Permisos de Windows | Ejecutar como administrador |

### 7.2 Estrategia de Reintentos
- 3 reintentos por video.
- Espera exponencial: 30s, 60s, 120s.
- Si falla → marcar como `Error` y continuar con el siguiente.
